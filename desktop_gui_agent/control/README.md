# 桌面控制模块 — 需求分析

> 参考：PDF 4.2 节（桌面控制模块）

---

## 模块职责

"操控屏幕"——接收来自 Agent 决策层的结构化动作指令，转化为真实的鼠标/键盘输入，执行桌面 GUI 自动化操作。

---

## 子模块

```
鼠标控制 (mouse_controller.py) ──→ 坐标移动 / 点击 / 拖拽
                                         │
键盘控制 (keyboard_controller.py) ──→ 文本输入 / 快捷键 / 滚动
                                         │
                              操作异常处理（统一返回状态）
```

---

## 4.2.1 鼠标控制 (mouse_controller.py)

**功能：** 实现屏幕坐标系的鼠标操作，模拟人工操作节奏。

| 操作 | 签名 | 说明 |
|------|------|------|
| 移动 | `move_to(x, y)` | 移动鼠标到指定坐标 |
| 左键单击 | `click(x=None, y=None, button='left')` | 移动并点击；x/y 为 None 时在当前位点击 |
| 右键单击 | `right_click(x=None, y=None)` | 移动并右键点击 |
| 双击 | `double_click(x=None, y=None)` | 移动并双击左键 |
| 拖拽 | `drag_from_to(x1, y1, x2, y2, duration=0.5)` | 从起点拖到终点，duration 控制拖拽时长 |

| 输入 | 类型 | 说明 |
|------|------|------|
| 坐标参数 | `int` | 以屏幕左上角为原点 (0,0) 的像素坐标 |

| 输出 | 类型 |
|------|------|
| 操作结果 | `bool`（True 成功，False 失败） |

**业务规则：**
- 坐标原点为屏幕左上角 (0, 0)
- 操作之间加入随机延迟（50~200ms），模拟人类操作节奏
- 默认使用适中的移动速度，避免瞬间跳跃
- 支持多屏幕环境下的绝对坐标

**异常处理：**

| 场景 | 处理 |
|------|------|
| 坐标越界（负数或超出屏幕） | 返回 `False`，记录警告日志 |
| 权限不足（无管理员权限） | 返回 `False`，记录错误日志 |
| 操作执行失败 | 返回 `False`，记录详细日志 |

## 4.2.2 键盘控制 (keyboard_controller.py)

**功能：** 实现键盘输入与组合键操作，模拟人工打字节奏。

| 操作 | 签名 | 说明 |
|------|------|------|
| 输入文本 | `type(text)` | 逐字符输入文本 |
| 按下按键 | `press(key)` | 按下并释放单个按键 |
| 按下（不释放） | `key_down(key)` | 按下按键不释放（用于组合键） |
| 释放按键 | `key_up(key)` | 释放之前按下的按键 |
| 组合键 | `hotkey(*keys)` | 依次按下所有键，再反向释放，如 `hotkey('ctrl', 'c')` |
| 滚轮滚动 | `scroll(direction, steps)` | direction 为 `'up'` 或 `'down'`，steps 控制滚动量 |

| 输入 | 类型 | 说明 |
|------|------|------|
| `text` | `str` | 要输入的文本，支持中英文 |
| `key` | `str` | 按键名称（如 `'ctrl'`, `'enter'`, `'a'`） |
| `direction` | `str` | `'up'` 或 `'down'` |
| `steps` | `int` | 滚动步数，每步约 120 像素 |

| 输出 | 类型 |
|------|------|
| 操作结果 | `bool`（True 成功，False 失败） |

**业务规则：**
- 支持中英文混合输入
- 支持常见系统快捷键（Ctrl+C、Win+R 等）
- 字符间随机延迟（30~100ms），模拟人工打字速度
- 组合键按下后确保正确释放，避免按键卡住

**异常处理：**

| 场景 | 处理 |
|------|------|
| 无效按键名称 | 返回 `False`，记录警告日志 |
| 输入文本为空 | 直接返回 `True`，不执行操作 |
| 权限不足 | 返回 `False`，记录错误日志 |
| 操作执行失败 | 返回 `False`，记录详细日志 |

---

## 4.2.3 操作异常处理

**功能：** 统一处理鼠标/键盘操作执行过程中的异常，确保 Agent 不会因单次操作失败而崩溃。

**设计原则：**
- **不抛出异常**：所有操作统一返回 `bool` 状态，由 Agent 决策层判断是否重试
- **详细日志**：每次失败时记录操作名、参数、异常类型和堆栈信息
- **权限感知**：区分权限不足（需用户介入）和临时失败（可重试）

| 异常类型 | 处理方式 | 返回值 |
|----------|----------|--------|
| 坐标越界 | 记录警告 + 裁剪提示 | `False` |
| 权限不足 | 记录错误 + 提示获取权限 | `False` |
| 设备未连接 | 记录错误 + 提示检查设备 | `False` |
| 临时执行失败 | 记录错误 + 建议重试 | `False` |
| 未知异常 | 记录完整堆栈 | `False` |

## 模块数据流

```
Agent 决策层
      │
      │  action: { action_type: "click", params: {x: 100, y: 200} }
      ▼
┌─────────────────────────────────┐
│       mouse_controller.py       │
│  move_to / click / drag / ...   │──→ 屏幕坐标操作
└─────────────────────────────────┘
      │
      │  action: { action_type: "type", params: {text: "Hello"} }
      ▼
┌─────────────────────────────────┐
│     keyboard_controller.py      │
│  type / hotkey / scroll / ...   │──→ 键盘输入操作
└─────────────────────────────────┘
      │
      ▼
  统一返回: {"success": True/False, "error": "..."}
      │
      ▼
Agent 决策层（根据结果决定重试/继续/报错）
```
