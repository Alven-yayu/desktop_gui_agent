# 桌面Agent决策模块 — 需求分析

> 参考：PDF 4.3 节（Agent决策模块）

---

## 模块职责

"做决策"——接收用户的自然语言任务，在每个步骤中：观察当前截图 → 调用视觉模型分析 → 解析出结构化动作 → 交给控制层执行，循环至任务完成或触发上限。

---

## 子模块

```
用户任务
      │
      ▼
model_client.py (模型客户端) ──→ 加载 Qwen2-VL，输入截图+任务描述，输出原始响应文本
      │
      ▼
action_parser.py (动作解析器) ──→ 将模型输出解析为结构化动作指令
      │
      ▼
task_manager.py (任务管理器) ──→ 主循环：截图→感知→模型→解析→执行，含安全机制
```

---

## 4.3.1 模型客户端 (model_client.py)

**功能：** 加载 Qwen2-VL 视觉语言模型，将截图和任务描述一起发送给模型，返回模型的原始文本响应。优先使用本地 Transformers 推理，同时支持 API 调用作为备选。

| 输入 | 类型 | 说明 |
|------|------|------|
| `image` | `PIL.Image` | 当前屏幕截图 |
| `task` | `str` | 用户的自然语言任务描述 |
| `context` | `list[str]`（可选） | 前几步的历史动作记录，帮助模型理解当前进度 |

| 输出 | 类型 | 说明 |
|------|------|------|
| 模型响应 | `str` | 模型的原始文本输出，预期包含动作指令 |

**业务规则：**
- 默认模式：本地加载 Qwen2-VL-2B-Instruct（Transformers），支持 CUDA 加速
- 备选模式：通过 API 调用远程 Qwen2-VL 服务（配置中切换）
- 每步把当前截图 PIL.Image 转成模型可接受的格式后传入
- Prompt 模板：系统提示词 + 任务描述 + 历史动作 + 截图 → 拼接后送入模型
- 支持指定 `max_tokens` 控制响应长度，避免模型输出无限文本
- 单次推理目标耗时 ≤ 3s（GPU），≤ 8s（CPU 回退）

**Prompt 模板（核心配置）：**

```
系统提示词：
你是桌面GUI智能体。根据截图和任务，输出下一步操作。

有效动作：
- click(x=<int>, y=<int>)           # 点击指定坐标
- type(text="<str>")                 # 输入文本
- scroll(direction="up|down", steps=<int>)  # 滚动
- hotkey(key1, key2, ...)            # 组合键
- finish(result="<str>")             # 任务完成

用户任务：{task}
历史动作：{context}
```

**异常处理：**

| 场景 | 处理 |
|------|------|
| 模型加载失败 | 抛 `ModelError`，记录详细错误日志 |
| 推理超时 | 抛 `ModelError`，记录推理耗时 |
| API 调用失败（网络/认证） | 重试 1 次，仍失败则抛 `ModelError` |
| 模型返回空文本 | 返回空字符串 `""`，由 action_parser 处理 |
| CUDA 不可用 | 自动回退 CPU，记录警告日志 |

## 4.3.2 动作解析器 (action_parser.py)

**功能：** 接收模型原始文本输出，解析为结构化的动作字典。支持正则匹配 + 模糊容错，确保即使模型输出格式不完美也能尽可能提取有效动作。

| 输入 | 类型 | 说明 |
|------|------|------|
| `model_output` | `str` | 模型原始文本响应 |

| 输出字段 | 类型 | 说明 |
|----------|------|------|
| `action_type` | `str` | 动作类型：`click` / `type` / `scroll` / `hotkey` / `finish` |
| `params` | `dict` | 动作参数，随类型不同 |

**五种动作的 params 格式：**

| action_type | params | 示例 |
|------|------|------|
| `click` | `{"x": int, "y": int}` | `{"x": 500, "y": 320}` |
| `type` | `{"text": str}` | `{"text": "Hello World"}` |
| `scroll` | `{"direction": str, "steps": int}` | `{"direction": "up", "steps": 3}` |
| `hotkey` | `{"keys": list[str]}` | `{"keys": ["ctrl", "c"]}` |
| `finish` | `{"result": str}` | `{"result": "已成功打开记事本"}` |

**业务规则：**
- 正则优先：先用严格正则匹配 `action_type(...)` 格式
- 模糊回退：正则失败时尝试从文本中提取关键词 + 坐标数字
- 参数校验：click 坐标必须为整数且在屏幕范围内（由 task_manager 校验）
- 无法解析时返回 `unknown` 类型，含原始文本，供 task_manager 重试
- 支持大小写不敏感匹配（如 `Click` / `CLICK` 均有效）
- 如果模型同时输出多个动作行，只取第一个有效动作（单步单动作原则）

**异常处理：**

| 场景 | 处理 |
|------|------|
| 输入为 `None` 或空字符串 | 返回 `{"action_type": "unknown", "raw": ""}` |
| 无法匹配任何动作格式 | 返回 `{"action_type": "unknown", "raw": model_output}` |
| 参数缺失（如 click 缺坐标） | 返回 `unknown`，记录原始文本 |
| 参数类型错误（如坐标是字符串） | 尝试转换，失败则返回 `unknown` |

## 4.3.3 任务管理器 (task_manager.py)

**功能：** 驱动整个 Agent 的主循环，每步依次执行"截图 → OCR 感知 → 模型决策 → 动作解析 → 操作执行"，直到任务完成、达到上限、或触发错误阈值。

| 输入 | 类型 | 说明 |
|------|------|------|
| `task` | `str` | 用户自然语言任务描述 |
| `max_steps` | `int`（可选，默认 20） | 最大步数上限 |
| `max_consecutive_errors` | `int`（可选，默认 3） | 连续错误次数阈值，超限则终止 |

| 输出 | 类型 | 说明 |
|------|------|------|
| 任务结果 | `dict` | `{"success": bool, "result": str, "steps": int, "error": str}` |

**主循环流程：**

```
步骤 0: 接收用户任务
          │
          ▼
┌────────────────────────────────┐
│  步骤 N (循环)                  │
│  1. capture() 截图              │
│  2. recognize() OCR 识别       │
│  3. client.query(image, task)  │
│  4. parser.parse(output)       │
│  5. 执行动作 (mouse/keyboard)   │
│  6. 收集结果 + 记录历史          │
│  7. 判断：finish? 达到上限?     │
│     - finish → 返回成功结果     │
│     - 达到上限 → 返回达到上限   │
│     - 否则 → 回到 1             │
└────────────────────────────────┘
```

**业务规则：**
- 单步单动作：每步只执行一个动作，不批量处理
- 步骤间延迟：每步之间加入 0.5~2s 随机延迟，等待 UI 响应
- 历史记录：每步记录（步骤号、截图路径、OCR 结果、模型输出、动作、执行结果）
- 坐标校验：click 坐标超出当前屏幕分辨率时自动跳过，记录警告
- 日志路径：`logs/` 目录下按日期保存，每步截图保存至 `logs/screenshots/`
- `max_steps` 默认 20，用户可通过 CLI `--max-steps` 自定义
- 连续错误阈值默认 3：连续 N 次（模型无输出 / 解析失败 / 操作失败）则终止

**异常处理：**

| 场景 | 处理 |
|------|------|
| 截图失败 | 记录错误，计入连续错误计数，跳过本步 |
| OCR 失败（引擎崩溃） | 记录错误，不计入连续错误（OCR 失败不影响继续） |
| 模型返回空 / 解析失败 | 记录警告，计入连续错误计数 |
| 操作执行返回 `False` | 记录警告，计入连续错误计数 |
| 连续错误达到阈值 | 终止循环，返回 `{"success": False, "error": "连续错误次数超限"}` |
| 步数达到上限 | 终止循环，返回 `{"success": False, "error": "达到最大步数上限"}` |
| 用户中断（Ctrl+C） | 优雅退出，保存已执行的步骤日志 |

## 模块数据流

```
用户任务 (str: "打开记事本输入Hello")
      │
      ▼
┌─────────────────────────────────────────────────────┐
│              task_manager.py 主循环                   │
│                                                      │
│  ┌──────────┐    ┌──────────┐    ┌───────────────┐  │
│  │ 1. 截图   │───→│ 2. OCR   │───→│ 3. 模型客户端  │  │
│  │ capture  │    │ recognize│    │   query()     │  │
│  └──────────┘    └──────────┘    └───────┬───────┘  │
│                                           │          │
│                                    model_output     │
│                                           │          │
│                               ┌───────────▼───────┐  │
│                               │ 4. 动作解析器      │  │
│                               │   parse()         │  │
│                               └───────────┬───────┘  │
│                                           │          │
│                            action: {action_type,     │
│                                     params}          │
│                                           │          │
│  ┌──────────┐                            │          │
│  │ 6. 记录  │←──── 执行结果 ──────────────┘          │
│  │ 历史/日志│                                       │
│  └────┬─────┘                                       │
│       │                                              │
│       ▼                                              │
│  action_type == "finish"?                            │
│       │                    ┌─────────────┐           │
│       ├── 是 ──→ 返回 success=True, result     │
│       │                    └─────────────┘           │
│       ├── 步数 >= max_steps?                         │
│       │   └── 是 ──→ 返回 success=False       │
│       └── 连续错误 >= 阈值?                          │
│           └── 是 ──→ 返回 success=False       │
└─────────────────────────────────────────────────────┘
      │
      ▼
最终输出: {"success": True, "result": "已打开记事本并输入Hello", "steps": 5}
```

## 新增配置项

添加到 `desktop_gui_agent/config.py`：

| 配置项 | 类型 | 默认值 | 说明 |
|------|------|------|------|
| `MODEL_NAME` | `str` | `"Qwen/Qwen2-VL-2B-Instruct"` | 本地模型名称或路径 |
| `MODEL_MODE` | `str` | `"local"` | 推理模式：`"local"` 或 `"api"` |
| `MODEL_API_URL` | `str` | `None` | API 端点（仅 API 模式） |
| `MODEL_API_KEY` | `str` | `None` | API 密钥（仅 API 模式） |
| `MODEL_MAX_TOKENS` | `int` | `512` | 单次推理最大输出 token 数 |
| `AGENT_MAX_STEPS` | `int` | `20` | 默认最大步数 |
| `AGENT_MAX_CONSECUTIVE_ERRORS` | `int` | `3` | 连续错误阈值 |
| `AGENT_STEP_DELAY` | `tuple` | `(0.5, 2.0)` | 步骤间随机延迟范围 (min, max)，秒 |
